Auth Sequence & Implementation Plan

UML sequence diagram: two sequential claims calls during token creation — legacy cdp_claims (org context, unchanged) followed by new cdp_app_claims (application registry roles). Both live as separate JSON claims in the JWT.

// ── EXISTING — unchanged ──────────────────────
if (features.Value.ClaimsApiEnabled)
{
    // … existing cdp_claims call … (line 143–168)
}

// ── NEW — AppRegistry application claims ──────
if (features.Value.AppRegistryClaimsEnabled)
{
    try
    {
        var encodedUrn = Uri.EscapeDataString(urn);
        using var resp = await httpClient
            .GetAsync($"/api/claims/{encodedUrn}");

        if (resp.IsSuccessStatusCode)
        {
            var json = await resp.Content.ReadAsStringAsync();
            claims.Add(new Claim(
                ClaimType.CdpAppClaims,
                json,
                JsonClaimValueTypes.Json));

            logger.LogDebug(
                "Added cdp_app_claims for {Urn}", urn);
        }
        else
            logger.LogWarning(
                "AppRegistry claims returned {Status} for {Urn}",
                resp.StatusCode, urn);
    }
    catch (Exception ex)
    {
        logger.LogWarning(ex,
            "Failed to fetch AppRegistry claims for {Urn}", urn);
    }
}

ClaimService.cs
CO.CDP.Authentication · ClaimService.cs
Add GetAppRegistryClaims() that reads from cdp_app_claims. Update HasApplicationRole() and HasApplicationPermission() to call it. The existing GetApplicationClaims() is left reading cdp_claims for backward compatibility.
// NEW — reads cdp_app_claims
public UserClaims? GetAppRegistryClaims()
{
    var claim = _httpCtx.HttpContext?
        .User.FindFirst(ClaimType.CdpAppClaims);
    if (claim?.Value is null) return null;
    return JsonSerializer.Deserialize<UserClaims>(claim.Value);
}

ClaimsEndpoints — Service Account Auth
Organisation.WebApi · ClaimsEndpoints.cs
The GET /api/claims/{userPrincipalId} endpoint currently requires PlatformAdmin (a human platform_role=admin claim). The Authority service calls it using a service account JWT. Either extend the PlatformAdmin policy or add service account channel auth — matching the pattern the legacy endpoint already uses.
// Option A — add ServiceAccount channel auth

app.MapGet("/api/claims/{userPrincipalId}", ...)
